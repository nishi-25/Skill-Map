ユーザー管理

「管理」→「ユーザー管理」から、全ユーザーの管理を行います。

新規ユーザーの承認

1ユーザー管理ページで「未承認」のユーザーを確認します。バッジに承認待ち件数が表示されます。
2「承認」ボタンをクリックしてユーザーを有効化します。

ロール変更

1ユーザー一覧でロールを変更したいユーザーを見つけます。
2「Manager に変更」または「User に変更」ボタンでロールを切り替えます。
注意: Admin ロールへの変更は画面から行えません。セキュリティのため、DBの直接操作が必要です。

スキル承認

「承認」→「承認依頼」から、ユーザーのスキル申告を確認・承認します。

1承認待ちの申告一覧を確認します。サイドバーにバッジで件数が表示されます。
2内容を確認し、「承認」または「却下」をクリックします。
3必要に応じてコメントを入力します（却下時は理由を記載することを推奨）。
ヒント: Admin と Manager 自身のスキル申告は自動承認されます。

スキルカタログ管理

「カタログ管理」→「スキルカタログ」から、ユーザーが申告できるスキル項目を定義します。

スキルの追加

1「新規スキル追加」ボタンをクリックします。
2スキル名、カテゴリー、ティア（難易度）、説明を入力します。
3「保存」をクリックしてスキルを登録します。

ティア（難易度レベル）

ティアはスキルの難易度を表す分類です。組織に合わせて名称をカスタマイズできます。

カテゴリー管理

「カタログ管理」→「カテゴリー」から、スキルのカテゴリーを管理します。

  • カテゴリーはスキルの分類に使用されます（例: プログラミング言語、フレームワーク、ツールなど）
  • カテゴリーの追加・編集・削除が可能です
  • ダッシュボードのレーダーチャートはカテゴリー単位で表示されます

グループ管理

「グループ」→「グループ管理」から、チーム / プロジェクトグループを管理します。

グループの作成

1「新規グループ作成」をクリックします。
2グループ名、説明、担当スキルを設定します。
3必要に応じて親グループを設定します。親グループのスキルが自動的に継承されます。

メンバー管理

1グループ詳細ページでメンバーの追加・削除を行います。
2メンバーは複数のグループに所属できます。

親グループ（階層構造）

グループは親子関係を持つことができます。子グループは親グループのスキルを自動的に継承します。

例: 「開発部」を親グループにし、「フロントエンドチーム」「バックエンドチーム」を子グループに設定すると、開発部の共通スキルが両チームに適用されます。

スキルマトリクス

「分析」→「スキルマトリクス」で、メンバーのスキル状況をマトリクス形式で一覧表示します。

フィルタリング

  • グループ: 特定のグループのメンバーのみ表示
  • カテゴリー: 特定カテゴリーのスキルのみ表示
  • ティア: 特定の難易度のスキルのみ表示

分析チャート

  • 成長トレンド: 月次の平均レベル推移グラフ
  • ユーザー別平均: メンバーのスキル平均ランキング
  • カテゴリー別レーダー: カテゴリーごとのスキルバランス
  • レベル分布: 全体のレベル分布
Manager の場合: 担当グループのメンバーのみが表示されます。Admin は全メンバーを閲覧できます。

管理ダッシュボード

Admin / Manager がログインすると、管理概要ダッシュボードが表示されます。

表示される情報

  • 全体サマリー: メンバー数、申告スキル数、平均レベル
  • グループ別スキル状況: 各グループの平均レベルと申告数
  • ティア別進捗: 難易度ごとの達成状況
  • メンバーランキング: 平均レベル順のメンバー一覧
  • 成長速度ランキング: 直近30日のレベルアップ数によるランキング
  • 最近の申告・更新: 直近の変更履歴

表示モードの切替

ダッシュボード上部のドロップダウンで表示モードを切り替えられます。

  • 全体概要: 全メンバーの集計
  • グループ: 特定グループのメンバーのみ
  • 個人: 特定メンバーの詳細

メール設定

「管理」→「メール設定」から、パスワードリセットメール等の送信に使用する SMTP 設定を行います。

設定項目

項目説明例
SMTP ホストメールサーバーのアドレスsmtp.gmail.com
SMTP ポートメールサーバーのポート番号587
SMTP ユーザー認証用のメールアドレスexample@gmail.com
SMTP パスワード認証用のパスワード / アプリパスワード—
差出人名メールの差出人として表示される名前スキルマップ
アプリURLパスワードリセットメール内のリンク先https://skillmap.example.com
Gmail の場合: 2段階認証を有効にし、「アプリパスワード」を発行して SMTP パスワードに設定してください。

パスワードリセット

メール経由のリセット（推奨）

メール設定が完了していて、ユーザーがメールアドレスを登録している場合は、ログイン画面の「パスワードをお忘れの方」からリセット可能です。

管理者による手動リセット（Docker コマンド）

メールが使えない場合やメールアドレス未登録のユーザーに対して、Docker コンテナ内でスクリプトを実行してリセットします。

1サーバーに SSH でアクセスします。
2以下のコマンドでリセットスクリプトを実行します。
docker exec -it skillmap python -c "
import database, models, auth
db = next(database.get_db())
user = db.query(models.User).filter(models.User.username == 'ユーザー名').first()
if user:
    user.password_hash = auth.hash_password('新しいパスワード')
    db.commit()
    print(f'{user.username} のパスワードをリセットしました')
else:
    print('ユーザーが見つかりません')
"
3ユーザー名 と 新しいパスワード を対象ユーザーのものに置き換えてください。
重要: パスワードは6文字以上で設定してください。リセット後、ユーザーにパスワードを安全な方法で通知してください。

Admin パスワードのリセット

Admin アカウントのパスワードを忘れた場合も同様にリセットできます。

docker exec -it skillmap python -c "
import database, models, auth
db = next(database.get_db())
admin = db.query(models.User).filter(models.User.role == 'admin').first()
if admin:
    admin.password_hash = auth.hash_password('新しいAdminパスワード')
    db.commit()
    print(f'Admin ({admin.username}) のパスワードをリセットしました')
"

全ユーザーの一覧確認

docker exec -it skillmap python -c "
import database, models
db = next(database.get_db())
for u in db.query(models.User).all():
    print(f'ID:{u.id}  {u.username}  role={u.role}  approved={u.is_approved}  email={u.email or \"未設定\"} ')
"

Docker 運用

基本コマンド

操作コマンド
ビルドDOCKER_BUILDKIT=0 docker build -t skill-map:latest .
起動docker compose up -d
停止docker compose down
再起動docker compose restart
ログ確認docker logs skillmap --tail 50
コンテナに入るdocker exec -it skillmap /bin/sh

データの永続化

データベース（SQLite）とアバター画像は Docker ボリューム skillmap_data に保存されます。docker compose down ではデータは保持されます。

注意: docker compose down -v を実行するとボリュームが削除され、全データが失われます。

バックアップ

# データベースのバックアップ
docker cp skillmap:/app/data/skillmap.db ./backup_skillmap.db

# 復元
docker cp ./backup_skillmap.db skillmap:/app/data/skillmap.db
docker compose restart

アクセスURL

デフォルトでは http://localhost:8190 でアクセスできます。ポートは docker-compose.yml で変更可能です。

トラブルシューティング

  • docker logs skillmap でエラーログを確認してください。
  • ポート 8190 が他のプロセスで使用されていないか確認してください。
  • ビルド時に DNS エラーが出る場合は DOCKER_BUILDKIT=0 を付けてビルドしてください。

  • メール設定（SMTP）が正しく設定されているか確認してください。
  • Gmail の場合、アプリパスワードが正しいか確認してください。
  • ユーザーのメールアドレスが登録されているか確認してください。
  • 迷惑メールフォルダを確認してください。

アプリは起動時に自動でDBマイグレーションを実行します。エラーが出る場合は、ログを確認し、必要に応じてバックアップしてからコンテナを再作成してください。
